---

使用 `design-thinking` 工作流，通过共情、构思和快速原型设计创建深植于用户需求的解决方案。

## 什么时候使用

- 为人设计产品或功能
- 解决用户体验至关重要的问题
- 从用户研究或共情工作开始
- 需要从洞察推进到可测试的原型
- 重新构想现有体验

## 什么时候跳过

- 纯技术问题，没有用户交互
- 基础设施或纯后端的问题
- 时间不允许进行用户验证

:::note[前置条件]
- **已安装带 CIS 的 BMad Method** — Maya 是 Creative Intelligence Suite 的一部分
- **一个设计挑战** — 需要探索的问题或机会
- **用户上下文** — 现有研究或利益相关者信息（可选但有帮助）
:::

## 步骤

### 1. 加载 Maya

开始新对话并加载 Design Thinking Coach：

```
/cis-design-thinking
```

### 2. 定义你的挑战

Maya 会问你的设计挑战。围绕用户需求来描述：

**好的挑战：**
- "如何帮助用户在开始新项目时更有信心？"
- "重新设计移动端购物者的结账体验"
- "帮助小企业主理解他们的现金流"

**效果较差：**
- "做一个新仪表盘"（先入为主地给了方案）
- "修复慢 API"（技术问题，不是以用户为中心）

### 3. 走过五个阶段

Maya 引导你完成完整的设计思维流程：

| 阶段 | 目标 | 你要做什么 |
| ---- | ---- | ---------- |
| **共情** | 理解用户 | 分享洞察、创建人物画像、绘制共情地图 |
| **定义** | 界定问题 | 打造 POV 陈述，提出"How Might We"问题 |
| **构思** | 生成方案 | 先大胆发散，再聚焦到有希望的概念 |
| **原型** | 让想法具象化 | 创建能传达想法本质的粗糙原型 |
| **测试** | 用用户验证 | 规划如何获取真实反馈 |

:::tip[先发散，再收敛]
Maya 强调在缩小范围之前先扩展选项。不要急着跳到方案——让过程自然展开。
:::

### 4. 应用设计方法

在每个阶段，Maya 从她的 design-methods 库中选择合适的方法：

- **共情**：用户访谈、共情地图、旅程图
- **定义**：POV 填空、"How Might We"框架、问题陈述
- **构思**：头脑风暴、草图、故事板
- **原型**：纸质原型、角色扮演、绿野仙踪法
- **测试**：可用性测试计划、反馈捕获模板

## 你将获得

输出保存到 `_bmad-output/design-thinking-{date}.md`：

| 部分 | 内容 |
| ---- | ---- |
| **设计挑战** | 你界定的机会 |
| **观点陈述** | 以用户为中心的问题陈述 |
| **用户洞察** | 共情发现和人物画像 |
| **How Might We 问题** | 将问题重新定义为机会 |
| **解决方案概念** | 生成的想法及理由 |
| **原型设计** | 可测试的产出和模型 |
| **测试计划** | 如何与真实用户验证 |
| **迭代路线图** | 基于学习的下一步 |

## 示例

```text
你：/cis-design-thinking
Maya：告诉我你的设计挑战，朋友。
     我们在为谁设计？
你：我们需要重新设计分析工具的新用户引导流程。
     用户在设置过程中大量流失。
Maya：啊，第一印象！让我们从共情开始。
     我们对这些用户了解多少？
你：[分享用户研究、痛点]
Maya：[引导完成共情地图]
     现在让我们打造我们的观点陈述...
     "新分析师需要感到有能力，而不是困惑"
     [框架化 How Might We 问题]
     [生成解决方案概念]
     [创建低保真原型]
     [规划验证方案]
```

## 技巧

:::tip[最佳实践]
- **带上用户数据** — 即使是非正式的洞察也能让工作更有根基
- **原型要粗糙** — 在这个阶段，草图和角色扮演比像素更有效
- **拥抱失败** — 负面测试结果是宝贵的学习
:::

:::caution[避免常见错误]
- 不要在共情之前就开始解题 — Maya 会把你拉回来
- 不要爱上你的第一个想法 — 发散才能创造更好的选项
- 不要跳过原型 — 认为某个方案可行不等于它真的可行
:::

## 下一步

设计思维之后：

- 使用**叙事**（`/cis-storytelling`）打造用户故事
- 运用**创新策略**（`/cis-innovation-strategy`）评估商业可行性
- 运行**头脑风暴**（`/cis-brainstorm`）获取更多方案选项

## 提供上下文

为了获得最佳效果，通过 `--data` 标志提供用户研究：

```bash
workflow cis-design-thinking --data /path/to/user-research.md
```

Maya 会使用这些上下文将共情和定义阶段建立在真实的用户洞察之上。

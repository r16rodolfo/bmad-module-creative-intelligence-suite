---

使用 `innovation-strategy` 工作流，运用经过验证的战略框架发现颠覆性机会并设计商业模式创新。

## 什么时候使用

- 探索新的市场机会或转型
- 分析竞争威胁和颠覆潜力
- 为新项目设计商业模式
- 寻求可持续的竞争优势
- 评估某个想法是否有战略价值

## 什么时候跳过

- 增量型功能改进（改用头脑风暴）
- 纯技术优化
- 规则清晰的成熟市场

:::note[前置条件]
- **已安装带 CIS 的 BMad Method** — Victor 是 Creative Intelligence Suite 的一部分
- **市场上下文** — 行业、竞争对手或机会领域
- **战略心态** — 愿意质疑既有假设
:::

## 步骤

### 1. 加载 Victor

开始新对话并加载 Innovation Strategist：

```
/cis-innovation-strategy
```

### 2. 定义你的战略问题

Victor 会问你的创新挑战。用战略视角来描述：

**好的战略问题：**
- "我们市场中的颠覆性机会在哪里？"
- "这个想法可以用什么商业模式？"
- "我们如何创造未被争夺的市场空间？"
- "我们的可持续竞争优势是什么？"

**效果较差：**
- "我们应该开发什么功能？"（产品策略，不是创新策略）
- "我们怎么打败竞争对手 X？"（战术层面，不是战略层面）

### 3. 分析市场格局

Victor 引导你完成：

| 分析维度 | 你将探索什么 |
| -------- | ------------ |
| **市场力量** | 趋势、变化、新兴客户需求 |
| **竞争动态** | 谁在赢、为什么、差距在哪里 |
| **价值链** | 价值在哪里被创造和获取 |
| **假设** | 哪些行业"规则"可能被打破 |

### 4. 运用战略框架

Victor 从他的 innovation-frameworks 库中调用：

| 框架 | 用途 |
| ---- | ---- |
| **Jobs-to-be-Done** | 理解客户实际雇佣产品去做什么 |
| **蓝海战略** | 寻找未被争夺的市场空间 |
| **颠覆性创新** | 识别低端或新市场机会 |
| **商业模式画布** | 设计或重塑你的商业模式 |
| **价值链分析** | 找到在哪里获取价值 |

:::tip[Victor 的风格]
Victor 像一个国际象棋大师——大胆的断言、战略性的沉默、一针见血的简单问题切穿复杂性。
:::

### 5. 定义你的战略定位

基于分析，Victor 帮你阐明：

- **创新论点** — 你的核心战略赌注
- **商业模式** — 你将如何创造和获取价值
- **护城河** — 可持续的竞争优势
- **执行优先级** — 最重要的事

## 你将获得

输出保存到 `_bmad-output/innovation-strategy-{date}.md`：

| 部分 | 内容 |
| ---- | ---- |
| **战略问题** | 你的创新挑战 |
| **市场分析** | 力量、趋势、竞争格局 |
| **Jobs-to-be-Done** | 未被满足的客户需求 |
| **蓝海机会** | 未被争夺的市场空间 |
| **商业模式** | 价值主张和获取机制 |
| **竞争优势** | 可防御的护城河 |
| **战略路线图** | 优先级和执行路径 |

## 示例

```text
你：/cis-innovation-strategy
Victor：我们在探索什么战略机会？
你：我们想颠覆家庭安防市场。
Victor：有意思。让我问一下：人们"雇佣"
       家庭安防来做什么工作？
你：[讨论客户需求、恐惧、动机]
Victor：[运用 Jobs-to-be-Done 分析]
       我明白了。这个行业在功能和价格上竞争。
       但真正的工作是"安心，而不是监控"。
       如果我们在安心感上竞争呢？
你：[探索蓝海机会]
Victor：[引导商业模式设计]
       这就是你的优势：你卖的不是报警器。
       你卖的是好觉。这是一种完全不同的商业模式。
```

## 技巧

:::tip[最佳实践]
- **挑战行业假设** — "规则"往往只是习惯
- **聚焦商业模式，而非功能** — 创新活在价值创造的方式中
- **长远思考** — 可持续优势胜过短期收益
:::

:::caution[避免常见错误]
- 不要把策略和战术混淆 — Victor 会纠正你
- 不要忽略商业模式 — 产品本身不是创新
- 不要假设在位者的优势 — 颠覆来自边缘
:::

## 下一步

创新策略之后：

- 使用**头脑风暴**（`/cis-brainstorm`）生成执行层面的想法
- 运用**设计思维**（`/cis-design-thinking`）塑造用户体验
- 运行**叙事**（`/cis-storytelling`）打造你的战略故事

## 提供上下文

为了获得最佳效果，通过 `--data` 标志提供市场分析：

```bash
workflow cis-innovation-strategy --data /path/to/market-analysis.md
```

Victor 会使用这些上下文将战略分析建立在你的市场现实之上。

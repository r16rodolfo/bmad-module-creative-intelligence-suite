使用 `storytelling` 工作流，运用经过验证的叙事框架、情感心理学和平台适配能力打造引人入胜的故事。

## 什么时候使用

- 创建产品或品牌叙事
- 撰写路演材料或投资人演示
- 开发用户故事或案例研究
- 传达变革或愿景
- 打造营销内容或活动

## 什么时候跳过

- 纯技术文档
- 不需要叙事的简单事实报告
- 极短的格式（标题、标语）

:::note[前置条件]
- **已安装带 CIS 的 BMad Method** — Sophia 是 Creative Intelligence Suite 的一部分
- **故事目的** — 你为什么要讲这个故事
- **受众** — 谁需要听到它
:::

## 步骤

### 1. 加载 Sophia

开始新对话并加载 Storyteller：

```
/cis-storytelling
```

### 2. 定义你的故事目的

Sophia 会问你的故事。对意图要清晰：

**好的故事目的：**
- "说服投资人我们的市场机会巨大"
- "激励员工拥抱新方向"
- "帮助用户理解我们的产品如何改变他们的生活"
- "用平易近人的方式解释一个复杂的技术概念"

**效果较差：**
- "写一些关于我们产品的东西"（没有明确目的）
- "讲我们公司的故事"（太模糊）

### 3. 了解你的受众

Sophia 探索你在跟谁对话：

| 受众问题 | 为什么重要 |
| -------- | ---------- |
| **他们是谁？** | 决定语言、语气、引用 |
| **他们现在相信什么？** | 识别需要跨越的认知差距 |
| **你想让他们感受什么？** | 情感旅程设计 |
| **他们应该做什么？** | 行动号召的清晰度 |

### 4. 选择你的框架

Sophia 从 25 种叙事框架中选择：

| 框架 | 适合场景 |
| ---- | -------- |
| **英雄之旅** | 转变、克服障碍 |
| **StoryBrand** | 以客户为中心的营销 |
| **三幕式结构** | 经典叙事弧线 |
| **Before-After-Bridge** | 简单的问题-方案故事 |
| **Pixar Pitch** | 情感驱动、角色驱动的叙事 |
| **倒金字塔** | 新闻风格、影响优先 |

:::tip[Sophia 的声音]
Sophia 像一个大师级的吟游诗人——华丽、奇幻，每一句话都让你沉浸其中，被叙事深深吸引。
:::

### 5. 打造你的叙事

Sophia 引导你完成：

| 故事元素 | 你将创建什么 |
| -------- | ------------ |
| **钩子** | 抓住注意力的开头 |
| **角色** | 可共鸣的主角和对手 |
| **冲突** | 问题或张力 |
| **旅程** | 穿越挣扎的路径 |
| **解决** | 令人满意的结局 |
| **转变** | 世界如何因此改变 |

### 6. 适配平台

Sophia 为故事的载体量身定制：

- **路演材料** — 简洁、逐页叙事
- **博客文章** — 可扫读、有叙事弧线
- **视频脚本** — 带对话的视觉叙事
- **社交媒体** — 适合信息流的微叙事
- **邮件** — 个人化、直接的叙事

## 你将获得

输出保存到 `_bmad-output/story-{date}.md`：

| 部分 | 内容 |
| ---- | ---- |
| **故事框架** | 使用了哪种结构及原因 |
| **受众画像** | 故事面向谁 |
| **情感弧线** | 你希望他们经历的感受旅程 |
| **完整叙事** | 含生动细节的完整故事 |
| **角色发展** | 声音、动机、转变 |
| **平台适配** | 针对你选择的媒介格式化 |
| **影响计划** | 如何衡量效果 |

## 示例

```text
你：/cis-storytelling
Sophia：你好，旅人。我们要编织什么故事？
你：我们需要为路演材料讲述产品故事。
     我们的应用帮助人们管理焦虑。
Sophia：啊，一个崇高的使命。告诉我——谁在
       被这种焦虑所困扰，你的方案如何成为
       他们一直等待的英雄？
你：[解释用户的挣扎和解决方案]
Sophia：[选择英雄之旅框架]
       让我们塑造你的主角——一个焦虑
       阻碍他充分生活的人。
       [发展角色和情感赌注]
       [创建有张力和释放的叙事弧线]
       [为路演材料适配——逐页]
```

:::note[Sophia 的记忆]
Sophia 会在她的 sidecar 中记住你的故事偏好和过往叙事。她会学习你的声音，并在之前的故事基础上不断积累。
:::

## 技巧

:::tip[最佳实践]
- **从"为什么"开始** — 先有目的，再有情节
- **让它有人味** — 具体细节胜过抽象概念
- **找到普遍性** — 个人故事加上普世真理，共鸣最广
:::

:::caution[避免常见错误]
- 不要埋没重点 — 立刻抓住读者
- 不要列举功能 — 讲述生活如何改变
- 不要过度写作 — 每个字都必须值得存在
:::

## 提供上下文

为了获得最佳效果，通过 `--data` 标志提供品牌或产品上下文：

```bash
workflow cis-storytelling --data /path/to/brand-guidelines.md
```

Sophia 会使用这些上下文保持品牌声音和一致性。
